「마크의 마지막 수리」 중앙 진행 UX 개선 보고서

현재 프로토타입의 선택 요소·결과 화면·모바일 동선을 전수 목록화하고, 화면을 덧붙이는 방식이 아닌 하나의 중앙 진행 흐름으로 재구성하기 위한 설계 문서입니다.

현 상태 진단
질문 단계의 중앙 허브는 추가됐지만, 사건 개요·현장·동료 의견은 여전히 서로 다른 패널과 모달에 흩어져 있습니다. 따라서 사용자는 “지금 무엇을 눌러야 하는지”와 “이전 정보가 어디에 있는지”를 매 단계 다시 판단해야 합니다. 아래 개선안은 이 구조를 전체 단계에 적용하는 것을 목표로 합니다.

1. 현재 화면·버튼·결과 전수 목록

각 항목은 현재 구현 위치와 문제를 기준으로 분류했습니다. 동작을 삭제하기보다 중앙 진행 카드에 흡수합니다.

핵심 원칙

한 화면에는 현재 단계의 핵심 행동 1개와 보조 행동 최대 2개만 둡니다. 결과는 새로운 패널이나 페이지가 아니라, 선택한 버튼이 있던 중앙 영역에서 크게 전환되어 나타납니다. 이전 정보는 화면을 되돌리는 대신 ‘사건 기록’ 하나로 접근합니다.

5. 모바일 필수 규칙

  • 가로 700px 이하에서는 모든 선택 카드를 한 열로 배치한다.
  • 현재 단계 외의 좌우 패널과 중복 하단 버튼은 표시하지 않는다.
  • 모든 주요 버튼의 높이는 최소 54px, 텍스트는 두 줄까지 허용한다.
  • 현장 사진은 찌그러뜨리지 않고 가로 스크롤 또는 전체 보기 모달 중 하나로 통일한다.
  • 선택 결과는 화면 중앙에 표시하고, 닫은 뒤에도 ‘사건 기록’에서 즉시 다시 볼 수 있어야 한다.
  • 스크롤 이동은 선택한 카드 또는 결과 카드 위치로만 수행한다. 문서 최상단 이동은 금지한다.

6. 구현 우선순위 및 검수 기준

1차 · 구조기존 3열 레이아웃을 공통 중앙 진행 카드로 교체하고, 단계마다 보이는 버튼을 단일 목록으로 관리한다.
2차 · 결과단서·동료·집사·정답 결과를 같은 중앙 결과 카드 컴포넌트로 통일하고 사건 기록에 누적한다.
3차 · 모바일375px / 390px / 430px 폭에서 버튼 잘림, 이미지 비율, 모달 스크롤, 고정 요소 겹침을 각각 확인한다.
4차 · 전수 테스트아래 체크리스트의 모든 버튼을 실제 순서대로 눌러 단계 전환·기록·재진입·시간 증가를 확인한다.
배포 전 완료 조건: “현재 단계에서 무엇을 누를지 3초 안에 알 수 있는가”, “결과를 닫은 뒤 다시 찾을 수 있는가”, “375px 폭에서 가로 스크롤 없이 버튼 문구를 읽을 수 있는가” 세 가지를 통과해야 합니다.

7. 전수 검수 체크리스트

  • 시작 안내 → 사건 개요 → 현장으로 이동
  • 현장 단서 3개 각각 선택, 결과 확인, 단서 재열람
  • 동료 4명 각각 선택, 말풍선 재열람, 추천 질문을 투표실에 반영
  • 추천 질문 선택 / 직접 입력 / 3회 제한 / 개인 시간 증가
  • 집사 답변 결과 확인 후 사건 기록에서 재열람
  • 정답 발의 성공·실패 모두 결과 및 시간 변동 확인
  • 사건 기록에서 사건 개요·단서·동료·질문 기록 재확인
  • 375px·390px·430px 및 데스크톱에서 버튼·이미지·모달·스크롤 확인

14. 구현 단위와 완료 정의

A. 공통 중앙 셸상단 바, STEP 헤더, 중앙 무대, 행동 영역, 사건 기록 모달모든 STEP이 같은 DOM 구조·CSS 토큰을 사용한다.
B. 단계 마이그레이션S1~S5 콘텐츠와 버튼을 중앙 셸에 이관기존 좌우 패널/중복 버튼/별도 진행 버튼이 제거된다.

E. 반응형 QA375/390/430/768/1440 폭 캡처 및 행동 테스트아래 전수 시나리오가 모든 폭에서 통과한다.

최종 인수 기준

  • 각 STEP에서 사용자가 누를 수 있는 버튼은 주 행동 1개와 보조 2개 이하이다.
  • 선택 결과는 100% 중앙 무대 또는 중앙 모달에서 표시된다.
  • 이전 정보는 사건 기록 하나에서 모두 재확인할 수 있다.
  • 동료 의견을 다시 눌러도 AI 재호출·시간 비용이 없다.
  • 질문 모달을 닫았다 열어도 질문 기록·잔여 횟수·추천 질문이 유지된다.
  • 375px에서 모든 중요 버튼·닫기·결과 문구가 한 화면 폭 안에서 읽힌다.
