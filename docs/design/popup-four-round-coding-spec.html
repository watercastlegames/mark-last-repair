각 필드 2~220자, 종합 의견 18~500자. “알 수 없음”, 내부 ID, 게임 밖 용어, 공개 기록에 없는 폭력 수단 등을 거절한다. 검증을 통과한 결과만 state.hints에 저장한다.

10. 집사 AI 판정 계약

judgeQuestion(question, roundNo)은 전체 사건 확정 사실표와 엄격한 논리 규칙을 AI에 전달한다. 공개 라운드와 관계없이 질문의 진실 여부는 전체 사건 정답을 기준으로 판정하지만 플레이어 화면에는 카테고리 문장만 노출한다.

category화면 문장핵심 규칙
YES네, 그렇습니다.주어·목적어를 포함한 모든 내용이 참
NO아니요, 그렇지 않습니다.모든 내용이 거짓 또는 확정 사실과 반대
MAYBE그럴 수도 있습니다.복합 질문 일부만 참, 대명사 대상 불명확
IRRELEVANT중요하지 않습니다.사건 무관 또는 게임 호스트 집사의 사건 관여 전제

첫 판정이 MAYBE이면 재판정 전용 프롬프트로 다시 확인한다. Puter 실패·시간 초과·형식 오류가 계속되면 질문을 저장하지 않고 비용도 추가하지 않은 채 입력 화면으로 돌아간다.

16. 소스 빌드와 배포

번들 생성

npx --yes esbuild assets/js/app-round11.js \ --bundle \ --format=iife \ --target=es2020 \ --outfile=assets/js/game-round54.bundle.js

정적 배포 필수 파일

index.html, developer.html, assets/**, 공개 문서 docs/**. 런타임에 ASP·PHP·Node 서버 코드는 필요하지 않다. Firebase 규칙은 배포 서버 파일이 아니라 Firebase Console에 별도로 게시한다.

캐시 무효화

CSS나 번들 변경 시 index.html의 query version을 올린다. 파일명과 query가 다를 수 있으므로 배포 후 실제 Network 응답 내용까지 확인한다.

서비스 조건

HTTPS 권장, Firebase 승인 도메인에 운영 hostname 등록, Puter.js와 Google Firebase CDN 접근 허용이 필요하다.
