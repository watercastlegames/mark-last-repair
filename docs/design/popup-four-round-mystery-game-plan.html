2. 현행 UX 원칙

중앙 집중: 메인화면은 사건명, 라운드 진행도, 동료, 시작·이어하기만 보여준다. 실제 선택과 결과는 중앙 팝업 하나에서 진행한다.

한 번에 한 행동: 사진 확인 → 단서 선택 → 동료 의견 → 질문 → 결과 → 라운드 정리 순으로 다음 행동을 명확하게 제시한다.

정보의 점진 공개: 미래 라운드의 인물·과거 사건·해답은 먼저 공개되지 않는다. 동료에게도 플레이어가 실제로 연 정보만 전달한다.

비용의 사전 확인: 질문 +1분, 시간의 방 +2분은 실행 직전 확인 화면에서 다시 알린다.

재열람 무료: 이미 공개된 사진, 단서, 시간의 방 정보, 집사 답변과 동료 의견은 비용 없이 다시 확인한다.

7. AI 동료 의견

최예선

차분하게 눈앞의 단서를 정리하고 연결 맥락을 한 단계씩 확인한다.

콩진호

확신을 서두르지 않고 사망 원인과 가능성을 조심스럽게 좁힌다.

남출구

가장 이상한 현장 요소를 거침없고 짧게 바로 묻는다.

이성민

경험 많은 전략가처럼 시간 비용 대비 효율적인 질문 순서를 강조한다.

동료 응답 계약

Puter AI가 ROUND 1부터 4까지 매번 추천 질문 한 문장과 짧은 의견을 생성한다. 질문은 물음표로 끝나며 한 가지 사실만 묻는다.

AI에게는 현재 라운드까지의 공개 단서, 실제 획득한 시간의 방 정보, 지금까지의 집사 질문·답변만 전달한다. 이미 들은 의견은 그대로 보존되며, 새 정보는 아직 의견을 듣지 않은 동료에게만 반영된다.

동료 이름·내부 ID·집사를 사건 인물로 사용하거나 미래 단서와 정답을 만들면 검증에서 거절하고 최대 세 번 재생성한다.

8. AI 집사 질문 판정

판정표시 문장사용 조건
YES네, 그렇습니다.질문의 모든 내용이 사건 확정 사실과 일치
NO아니요, 그렇지 않습니다.질문의 모든 내용이 확정 사실과 반대
MAYBE그럴 수도 있습니다.복합 질문의 일부만 맞거나 대명사·대상이 불명확
IRRELEVANT중요하지 않습니다.사건 해결과 무관하거나 집사를 사건 인물로 전제
표현 원칙: 집사는 판정 문장만 말하고 보충 설명을 하지 않는다. 질문 선택만으로는 비용이 없고, 확인 팝업에서 확정한 뒤에만 질문 수 +1과 개인 시간 +1:00을 적용한다. AI 연결 실패 시 질문 수와 시간은 증가하지 않는다.
재판정: 첫 AI 판정이 MAYBE이면 같은 사건 사실표로 한 번 더 엄격하게 확인한다. 주어와 대상이 명확해 YES 또는 NO가 가능한 질문에 MAYBE를 기본값처럼 사용하지 않는다.

로컬 최고 기록점수 → 질문 수 → 시간 순으로 이 기기의 최고 기록 저장
공식 랭킹Firebase 데이터 기준 점수 내림차순 → 질문 수 오름차순 → 공식 시간 오름차순, 상위 100명
계정당 1회결과 화면에서 Google 로그인 후 동일 사건·동일 UID 문서를 최초 한 번만 생성. 수정·삭제 불가
신뢰 수준: 현재 공식 기록의 시간·질문 수·점수는 브라우저에서 계산해 Firestore 규칙의 범위 검증을 거쳐 저장한다. 별도 권위 서버나 부정행위 탐지 시스템은 없다. 따라서 “공개 팬게임 랭킹” 수준이며 e스포츠·보상 지급용 검증 기록으로 사용하지 않는다.

12. 진행 저장과 새로고침 복구

자동 저장 대상: 현재 phase, 라운드, 공개 단서, 질문·답변, 시간 페널티, 시작·완료 시각, 시간의 방 방문 수, 동료 의견, 최종 발의 초안.

복구: 새로고침하면 저장된 단계의 중앙 팝업을 다시 연다. AI 요청 중이던 QUESTION_PENDING은 질문 선택 단계로 되돌려 이중 비용을 막는다.

보존 범위: 같은 브라우저의 localStorage. 다른 기기와 진행 상황은 동기화하지 않는다. Firebase에는 완료 후 공식 랭킹용 최소 결과만 저장한다.

13. 모바일 우선 UX

  • 모바일 HUD는 시간, 1위 기록, 사건 기록, 랭킹만 한 줄에 표시한다.
  • 팝업은 좌우 6px 여백, 최대 높이 100dvh - 58px, 내부 본문만 세로 스크롤한다.
  • 사진은 16:9 캔버스 안에서 object-fit: contain으로 원본 비율을 유지한다.
  • 팝업 버튼은 모바일에서 한 열이며 최소 높이 54px로 제공한다.
  • 동료 선택은 일반 모바일 2열, 370px 이하 1열로 바뀐다.
  • 말풍선은 모바일에서 초상화 아래로 이동하고 꼬리 방향도 위쪽으로 바뀐다.
  • 본문의 가로 오버플로를 숨기고 하단 safe area를 반영한다.
  • prefers-reduced-motion 사용자는 애니메이션 시간을 사실상 제거한다.

15. 현행 완료·회귀 테스트 기준

  • ROUND 1~4 각각 그림 1장, 단서 3개, 시간의 방 3개, 질문 최대 3회가 작동한다.
  • 단서 3개를 모두 열기 전 동료 진행 버튼이 비활성화된다.
  • 질문과 시간의 방 비용은 성공 시 한 번만 추가되고 실패 시 추가되지 않는다.
  • 동료 AI는 플레이어가 실제로 공개한 시간의 방 정보와 집사 답변을 포함한다.
  • 집사는 네 고정 문장만 표시하고 보충 설명을 출력하지 않는다.
  • 사건 기록에서 공개된 과거 사진·단서·시간의 방·질문을 다시 볼 수 있다.
  • 최종 동료 발의안을 채택하면 6개 입력칸이 보존된 상태로 채워진다.
  • 정답·부분정답·오답과 정답 기준이 결과 화면에 항목별로 표시된다.
  • 새로고침 후 현재 라운드와 수사 기록이 유지된다.
  • 390×844 모바일에서 가로 스크롤, 잘린 팝업, 찌그러진 사진이 없다.
  • Google 로그인 성공 후 최초 기록만 생성되고 두 번째 등록은 거절된다.
  • Firebase 실패 시 가짜 공식 랭킹을 만들지 않고 로컬 기록임을 명확히 표시한다.
