4. 사진과 현장 단서 3개

사진 위 물음표 3개와 하단 단서 카드 3개가 동일한 단서 ID에 연결된다. 모바일에서 물음표가 가려져도 하단 카드를 통해 같은 기능을 이용할 수 있다.

처음 누르면 clueIds에 저장하고 결과 팝업을 연다. 다시 누르면 “이미 확인한 단서”임을 알리고 추가 비용 없이 같은 내용을 보여준다.

진행 버튼은 0~2개일 때 비활성화하고 정확히 3개를 연 뒤 “동료 의견 →”으로 바뀐다.

5. 시간의 방

  1. 단서·동료·질문 단계의 보조 버튼에서 현재 획득 수/3을 표시한다.
  2. 입장 전 “새 정보 +2:00” 확인 팝업을 보여준다.
  3. 확정하면 페널티와 방문 수를 먼저 저장하고 새 정보 한 개를 공개한다.
  4. 정보는 사건 기록과 이후 처음 호출하는 동료의 공개 context에 포함된다.
  5. 세 개를 모두 얻은 뒤에는 목록을 무료 재열람한다.
  6. 돌아가기 목적지는 시간의 방을 연 원래 단계(단서·동료·질문)다.
난이도 설계: 정보 1은 방향, 정보 2는 연결, 정보 3은 핵심 사실을 강하게 제시한다. 사용자는 필요한 만큼만 시간을 지불한다.

8. 라운드 정리와 다음 단계

단서 3개, 시간의 방 방문 수, 이번 라운드 질문 수, 누적 시간을 2×2 요약 카드로 보여준다. ROUND 1~3은 “다음 그림 공개 →”, ROUND 4는 “최종 발의로 이동 →”을 주 행동으로 사용한다.

라운드 이동 시 roundNo를 1 증가시키고 현재 라운드 questionCount만 0으로 초기화한다. 전체 질문, 페널티, 공개 정보와 동료 의견은 유지한다.

10. 최종 작전 회의와 발의서

상단에 동료 4명을 다시 표시한다. 최종 의견을 처음 들을 동료에게는 공개 단서 수와 문답 수를 보여주고, 이미 들은 동료는 “최종 의견 다시 듣기”로 표시한다.

AI는 피해자·가해자·장소·사망 원인·동기·숨겨진 진실과 종합 의견을 생성한다. 공개 기록 밖의 사실은 금지하지만 불확실하더라도 가장 가능성 높은 가설 하나를 각 항목에 제안한다.

“이 의견으로 발의서 채우기”는 여섯 필드를 모두 동료안으로 교체한 뒤 최종 발의 화면으로 돌아온다. 이후 사용자가 자유롭게 수정할 수 있다.

입력 중 사건 기록이나 동료 의견을 열어도 초안은 유지한다. 여섯 칸 중 하나라도 비어 있으면 제출하지 않고 안내한다.

11. 사건 내막과 상세 결과

  1. 제출 즉시 완료 시각과 해결 여부를 저장한다.
  2. 사건 내막 01~04를 이미지, 이야기, 핵심 증거와 함께 순차 표시한다.
  3. 첫 내막에서 총점과 정답·부분정답·오답 개수를 미리 알려준다.
  4. 마지막 장면 뒤 결과 화면에서 여섯 항목의 내 발의, 정답 기준, 판정 이유를 모두 표시한다.
  5. 결과에는 총시간, 질문 수, 시간의 방 총 사용량을 표시한다.
  6. 전체 내막 재열람, 랭킹, Google 기록 등록을 제공한다.

14. 모바일 상호작용 검수 규격

  • 390×844 기준 팝업이 뷰포트 안에 있고 본문만 스크롤된다.
  • 팝업 미니 타이머는 노치 safe area 아래 중앙에 고정된다.
  • 모든 진행 버튼은 한 열, 최소 54px 높이이며 눌러야 할 주 버튼이 가장 아래에 명확하다.
  • 사진은 찌그러지거나 잘리지 않고 원본 비율을 유지한다.
  • 물음표는 최소 40×40px이며 하단 카드가 동일 기능을 보완한다.
  • 동료 카드를 누르면 별도의 동료 결과 팝업으로 전환되어 초상화와 말풍선이 화면 안에 보인다.
  • 질문 입력에서 키보드가 떠도 입력과 확인 버튼을 스크롤로 접근할 수 있다.
  • 최종 발의 6개 필드는 1열이며 입력 중 재렌더링으로 값이 사라지지 않는다.
  • 우측 하단 공유·개발자 버튼이 결과 버튼이나 팝업보다 높은 조작 우선권을 갖지 않는다.
  • 배경에는 가로 스크롤이 생기지 않는다.
