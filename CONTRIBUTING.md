# 기여 안내

버그 수정, 모바일 사용성 개선, 문서 보완과 새로운 미스터리 구조 제안을 환영합니다.

## 시작하기

1. 저장소를 Fork합니다.
2. 작업 목적이 드러나는 브랜치를 만듭니다.
3. `python -m http.server 8080`으로 로컬 서버를 실행합니다.
4. `http://localhost:8080/`에서 모바일과 PC 흐름을 확인합니다.
5. 변경 이유와 확인 방법을 Pull Request에 적습니다.

## Pull Request 확인 항목

- 한 질문에는 가능한 한 하나의 사실만 포함되는가?
- AI가 아직 공개되지 않은 정보를 동료 의견으로 말하지 않는가?
- 모바일 360px에서 가로 스크롤과 잘린 팝업이 없는가?
- 새로고침 후 진행 단계가 유지되는가?
- 이미지 경로와 대체 텍스트가 올바른가?
- 소스 모듈 수정 후 배포 번들을 다시 만들었는가?
- 비밀번호, 토큰, 서버 주소와 개인 정보가 포함되지 않았는가?

## 새로운 사건을 제안할 때

[새 테마 제작 가이드](docs/guides/CUSTOM_THEME_GUIDE.md)의 정답표와 네 라운드 구조를 먼저 작성해 주세요.

실제 인물을 범인이나 피해자로 묘사하거나, 방송 캡처·무단 사진·타인의 저작물을 포함하는 테마는 받지 않습니다. 가상 인물과 직접 제작하거나 적법하게 사용할 수 있는 에셋을 사용해 주세요.

## 코딩 방식

AI를 사용한 바이브코딩 기여도 환영합니다. 다만 결과를 그대로 제출하지 말고 실제 브라우저에서 전체 흐름을 한 번 이상 진행해 주세요. AI가 만든 사건 데이터와 판정 프롬프트 사이에 모순이 없는지 사람이 최종 확인해야 합니다.
