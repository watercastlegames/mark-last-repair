# 보안 정책

## 제보가 필요한 문제

- 다른 사용자의 랭킹 기록을 수정하거나 삭제할 수 있는 문제
- 같은 계정으로 사건 기록을 여러 번 생성할 수 있는 문제
- Firestore 규칙을 우회하는 읽기·쓰기
- 사용자 입력으로 스크립트가 실행되는 XSS
- 로그인 토큰이나 사용자 식별 정보 노출
- 서버 API의 인증 또는 사건 상태 검증 우회

공개 Issue에 실제 토큰, 사용자 정보나 재현용 계정을 올리지 마세요. 민감한 내용은 개발자 Threads 계정의 연락 수단을 통해 비공개로 먼저 알려 주세요.

## 공개 설정값

`assets/js/firebase-config.js`에 들어가는 Firebase Web API Key는 브라우저 앱에 배포되는 공개 식별자입니다. 이 값만 노출된 것은 보안 취약점이 아닙니다. Firestore 규칙 우회나 승인되지 않은 데이터 접근이 함께 가능할 때 제보해 주세요.

## 지원 범위

현재 공개 서비스에 배포된 최신 버전과 저장소 기본 브랜치를 우선 지원합니다.
