마크의 마지막 수리 — 4라운드 중앙 팝업형 추리 게임

메인화면은 진행 현황만 보여주고, 그림 공개·단서 선택·동료 의견·질문·판정·라운드 전환·최종 해설은 모두 화면 중앙의 몰입형 팝업에서 진행한다.

중앙 팝업 진행4라운드 × 질문 최대 3회동료 4명Google 로그인사건당 계정 1회서버 랭킹
사용자 의도 최종 정의“메인화면에서 여러 패널을 직접 돌아다니는 게임”이 아니다. 사용자가 현재 해야 할 행동 하나를 누르면 중앙 팝업이 열리고, 그 팝업 안에서 선택과 결과를 확인한다. 단계가 끝날 때마다 다음 단계 안내 팝업이 자동으로 뜬다. 이 순환을 네 번 진행한 후 최종 답안과 전체 사건 내막·성적·랭킹을 보여준다.

플레이 단위

1라운드는 그림 1장, 공개 단서 3개, 동료 의견, 집사 질문 최대 3개로 구성한다. 세 번째 질문 답변이 끝나면 자동으로 라운드 정리 팝업을 띄우고, “다음 그림 공개”를 눌러 다음 라운드로 이동한다.

플레이어는 질문을 최소 1개 한 뒤 “남은 질문 포기하고 다음 라운드”를 선택할 수 있다. 이 경우 사용 질문 수가 줄어 랭킹에 유리할 수 있지만, 포기한 질문은 되돌릴 수 없다. 따라서 전체 질문 수는 4~12회로 기록된다.

자동 전환 규칙: 질문 3회 완료 → 라운드 결과 팝업 자동 표시 → 다음 라운드 예고 → 다음 그림 팝업. 사용자가 메인화면에서 다음 버튼을 찾아다니지 않는다.

4. 한 라운드의 정확한 팝업 순서

  1. 라운드 시작 팝업 — “새로운 그림이 공개됩니다”와 현재 라운드 목표를 표시한다.
  2. 그림 공개 팝업 — 화면 중앙에 그림을 크게 표시한다. 모바일에서는 원본 비율을 유지하고 확대·가로 이동을 지원한다.
  3. 단서 선택 팝업 — 그림 속 물음표 또는 하단 단서 카드 3개 중 하나를 누른다.
  4. 단서 결과 팝업 — 선택한 단서 설명 한 문장만 표시하고 “다음 단서”를 누른다. 3개 완료 시 자동으로 동료 단계로 간다.
  5. 동료 선택 팝업 — 네 명 중 한 명을 선택한다. 동일 라운드에서 모두 들을 수 있고 재열람은 무료다.
  6. 동료 말풍선 팝업 — 동료가 이번 라운드 공개 정보만 근거로 질문 하나를 추천한다. “이 질문 사용” 버튼을 제공한다.
  7. 질문 선택 팝업 — 동료 추천 질문 또는 직접 입력을 고른다. 질문 확정 전 시간 비용은 없다.
  8. 질문 확인 팝업 — 질문 문장과 비용 +1:00을 보여주고 최종 확인을 받는다.
  9. 집사 판정 팝업 — 네 가지 허용 문장 중 하나만 크게 표시한다. 최대 3회까지 6~9를 반복한다.
  10. 라운드 완료 팝업 — 획득 단서, 사용 질문 수, 누적 시간을 요약하고 “다음 그림 공개”를 표시한다.

8. 최종 발의와 전체 내막 공개

  1. 최종 발의 예고 — 네 장의 그림과 핵심 기록이 차례로 쌓이는 연출을 보여준다.
  2. 정답 입력 — 피해자, 가해자, 장소, 동기, 사망 원인, 숨겨진 과거를 입력한다.
  3. 판정 — 맞으면 해결, 틀리면 미해결로 표시한다. 오답이어도 다음 해설을 막지 않는다.
  4. 전체 내막 — ROUND 1~4 그림을 한 장씩 다시 보여주며 각 장면이 의미했던 사실을 설명한다.
  5. 사건 타임라인 — 12년 전 금화 절도 → 마크의 기록 발견 → 그랜트의 위장 계획 → 배전실 감전 사망 순서로 공개한다.
  6. 개인 결과 — 정답 여부, 실제 플레이 시간, 시간 페널티, 사용 질문 수, 포기 질문 수, 최종 순위를 보여준다.

최종 해설의 핵심

그랜트는 12년 전 금화 절도와 연결되어 있었다. 마크가 관련 기록을 발견하자, 그랜트는 전기 사고처럼 보이도록 배전실 환경을 조작했다. 젖은 바닥과 차단기, 분장실의 스프레이 병, 과거 기록이 하나의 사건으로 연결된다. 실제 확정 내막은 콘텐츠 검수 후 서버 사건 데이터로 관리한다.

10. Google 로그인과 사건당 1회 제한

순서동작보안 기준
1사건 시작 전에 Google Identity Services가 공식 로그인 버튼을 렌더링한다.임의로 만든 Google 모양 버튼을 사용하지 않는다.
2로그인 팝업에서 받은 ID 토큰을 HTTPS로 서버에 전달한다.브라우저에서 디코딩한 값만 믿지 않는다.
3서버가 토큰 서명, aud, iss, exp를 검증한다.Google 계정 키는 이메일 대신 변경되지 않는 sub를 사용한다.
4서버가 (google_sub, case_id) 기존 시도를 조회한다.완료 상태면 재시작 차단, 진행 중이면 기존 세션 재개.
5새 시도라면 서버에 시작 시간을 기록하고 게임 세션 쿠키를 발급한다.HttpOnly, Secure, SameSite=Lax 세션 쿠키 권장.
6완료 시 서버 트랜잭션으로 결과와 랭킹 레코드를 확정한다.완료 요청 재전송에도 결과가 한 번만 생성되도록 멱등 처리.
중요한 기술 조건: 현재처럼 정적 HTML과 클라이언트 Puter 호출만으로는 계정당 1회 제한과 공정한 랭킹을 보장할 수 없다. Google 토큰 검증, 시도 잠금, 공식 시간 계산, 결과 저장을 담당하는 서버 API와 데이터베이스가 반드시 필요하다.

13. 중복 플레이 방지와 기록 신뢰성

  • 로컬스토리지나 쿠키만으로 재도전을 막지 않는다. 삭제·변조가 가능하므로 서버 고유 제약을 사용한다.
  • Google 이메일이 아니라 ID 토큰의 sub 값을 사용자 식별키로 사용한다.
  • 진행 도중 브라우저를 닫으면 같은 시도를 이어서 한다. 새 시도로 간주하지 않는다.
  • 완료된 사건은 같은 계정으로 다시 플레이할 수 없고, 최종 해설과 자기 기록만 다시 볼 수 있다.
  • 공식 시간은 클라이언트 타이머가 아니라 서버의 시작·이벤트·완료 시간을 기준으로 계산한다.
  • 질문 수, 라운드 진행, 페널티는 서버가 검증한다. 화면 개발자 도구로 값을 바꿔도 랭킹에는 반영되지 않는다.
  • 버전이 다른 사건은 별개의 case_id로 취급할 수 있으나, 운영자가 새 버전을 새 사건으로 공개할 때만 허용한다.

17. 최종 인수 테스트

  • 로그인하지 않으면 사건 시작과 기록 등록이 불가능하다.
  • ROUND 1 그림부터 ROUND 4 그림까지 중앙 팝업으로만 순서대로 진행된다.
  • 각 라운드에서 단서 3개, 동료 의견, 질문 최대 3개가 빠짐없이 제공된다.
  • 세 번째 질문 결과 직후 다음 라운드 안내 팝업이 자동으로 열린다.
  • 동료는 현재까지 공개된 정보만 말하며 추천 질문 버튼이 질문 팝업에 추가된다.
  • 질문·단서·동료 결과를 사건 기록에서 다시 볼 수 있다.
  • ROUND 4 종료 뒤 정답 여부와 관계없이 전체 내막과 네 장의 그림 해설을 본다.
  • 최종 결과에 정답 여부, 공식 시간, 페널티, 사용 질문 수, 내 순위가 표시된다.
  • 같은 Google 계정으로 같은 사건을 두 번째 시작할 수 없다.
  • 중단 후 재접속하면 새 게임이 아니라 기존 라운드와 기록을 복구한다.
  • 375px 모바일에서 팝업, 그림, 말풍선, 추천 질문, 결과 버튼이 잘리지 않는다.

18. Google 로그인 기술 참고

Google Identity Services는 공식 버튼과 팝업/리디렉션 로그인 흐름을 제공하지만, 웹사이트 자체 세션은 별도로 관리해야 한다. 로그인 후 받은 ID 토큰은 서버에서 검증하고, 사용자 식별에는 이메일이 아닌 sub를 사용한다.
